# Stitch Prompts

Use these prompts to generate the UI screens first. The visual target is a **serious civic operations dashboard**, not a startup landing page.

## Global Art Direction Prompt

Design a high-credibility traffic command dashboard called `LaneGuard` for Bengaluru Traffic Police. The product identifies illegal-parking hotspots that likely cause congestion and helps teams prioritize enforcement. Use a map-first layout, dark command-center canvas, bright cobalt and amber accents, strong hierarchy, large numeric KPI cards, crisp tables, and a polished government-tech feel. Avoid generic SaaS layouts, purple gradients, rounded toy cards, and marketing fluff. The UI should feel operational, dense, calm, and authoritative.

Important: include a visible layer for explainable intervention suggestions and policy decision support. Recommendations should look evidence-based, not magical.

## Screen 0: Mission Launchpad

Create the first screen of `LaneGuard` as a polished operational launchpad, not a developer workbench or internal scaffold. This is the first impression for judges and police stakeholders. It must feel like a real city-operations product home.

Include:

- strong hero headline focused on parking-pressure response and congestion relief
- quick-launch cards into the main modules: Command Center, Enforcement Planner, Policy Recommendations, Station Analytics, Data Explorer, Daily Brief
- a live geospatial preview panel with Bengaluru hotspot markers
- a compact KPI strip showing total parking violations, active hotspots, lead station, and peak pressure window
- a “judge demo flow” or “recommended sequence” panel that clearly guides the 2-minute walkthrough
- a priority stations panel showing the most affected jurisdictions
- a screen inventory or module readiness area that makes the product feel complete and deliberate

Visual direction:

- cinematic command-center mood, not a template dashboard
- asymmetrical layout with one bold hero zone and secondary information rails
- dense but readable composition with clear hierarchy
- use restrained motion-ready framing and premium surfaces
- avoid generic cards on a blank dark background
- make the page feel like a mission briefing deck merged with an operations console

Tone:

- authoritative
- public-sector credible
- presentation-ready
- visually memorable enough that the user immediately understands this is the control hub for the whole product

## Screen 1: Command Center

Create a desktop web dashboard for `LaneGuard` with these elements:

- full-height Bengaluru map at center-left
- hotspot markers in yellow to red severity scale
- top KPI cards across top: total violations, active hotspots, top station, peak hour
- right-side ranked "Intervene Now" panel with top 5 hotspots
- filter bar for station, time band, violation type, severity
- dark theme with sharp contrast and map-first emphasis
- footer legend for hotspot severity

The screen must look like a real operations center, not a BI template.

## Screen 2: Hotspot Detail Drawer

Design a slide-over detail panel for a selected hotspot in `LaneGuard`. Include:

- hotspot name / area
- priority score out of 100
- reason chips like "Recurring 9 days", "Main-road parking", "Peak-hour dense"
- mini time-series chart
- pie or bar chart for violation composition
- station ownership
- recommended action box
- grouped recommendation cards for Immediate, Short-term, and Medium-term actions
- rationale lines under each recommendation
- call-to-action button: `Add To Shift Plan`

The panel should feel precise, high-confidence, and explainable.

## Screen 3: Enforcement Planner

Design a desktop screen for `LaneGuard` called `Enforcement Planner`. Include:

- ranked hotspot table
- urgency tags
- shift selector
- station filter
- selected action queue on the right
- projected relief summary card
- map mini-preview

Focus on usability for a supervisor assigning scarce field resources.

## Screen 4A: Policy Recommendations

Design a policy decision-support screen for `LaneGuard`. Include:

- ranked list of intervention suggestions
- categories: Immediate, Short-term, Medium-term
- hotspot linkage
- reason chips tied to observed data
- confidence band or recommendation strength
- note that these are recommendations, not final civil decisions

The screen should feel evidence-led and operationally credible.

## Screen 5: Station Analytics

Design a data-rich analytics screen for `LaneGuard` comparing police stations on parking-induced congestion pressure. Include:

- sortable station comparison table
- bar chart of violations by station
- heatmap of peak hours by station
- repeated-hotspot trend chart
- compact map snippets

This should feel analytical but still operational.

## Screen 6: Daily Brief

Design a concise executive summary screen for `LaneGuard`. Include:

- headline: `Today's Priority Parking Pressure`
- total hotspots, top affected stations, peak time
- top 5 intervene-now cards
- projected relief panel
- top policy recommendations summary
- a compact city map with highlighted corridors

This should be presentation-ready for senior leadership.

## Screen 7: Data Explorer

Design a credible raw-record explorer screen for `LaneGuard` to support trust and auditability. Include:

- search
- filters
- record table
- violation chips
- station column
- timestamp column
- map preview for selected row

Use the same visual language as the rest of the system.
