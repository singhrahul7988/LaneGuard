# Demo Script

## Goal

Deliver a 2 to 3 minute walkthrough that feels like a real command-center intervention workflow.

## Script

### 1. Open With The Problem

"Parking violations are visible in raw records, but enforcement is still reactive. `LaneGuard` helps identify which parking hotspots are likely causing the most disruption and where officers should intervene first."

### 2. Show The Command Center

- Open the main map
- Point to hotspot density
- Read top KPI cards

Line:

"Instead of a spreadsheet, we now have a city-level operational view."

### 3. Drill Into One Hotspot

- Click a severe hotspot
- Show score reasons
- Show violation mix and time trend

Line:

"This hotspot is recurring across multiple days, concentrated in peak hours, and includes main-road and crossing-related violations. That makes it operationally higher risk."

### 3A. Show Suggested Actions

- Show recommendation cards
- Explain immediate vs short-term vs medium-term actions

Line:

"We do not stop at detection. LaneGuard also suggests explainable interventions based on the hotspot pattern, so the team knows what to do next."

### 4. Build A Shift Plan

- Add hotspot to planner
- Add 2 more
- Show ranked queue

Line:

"Now a station supervisor can convert intelligence into a shift-ready intervention plan."

### 5. Show Daily Brief

- Open Daily Brief
- Show top 5 hotspots and projected relief

Line:

"Leadership gets a one-screen summary of where to act today and why."

### 6. Close Strong

"LaneGuard turns real parking violation data into explainable enforcement intelligence for Bengaluru Traffic Police."

## Demo Rules

- Never start in a raw table
- Never spend more than 20 seconds on architecture
- Keep the city map visible as often as possible
- Use one compelling hotspot story, not ten weak ones
